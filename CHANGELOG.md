# Changelog

## [4.0.0] — 2025-XX-XX

### Изменения по сравнению с v3

#### Архитектура
- Бизнес-логика (парсинг URL, генерация конфигов) отделена от GUI — проще тестировать и переиспользовать.
- Введены dataclass-модели `VlessParams` и `ForcedRules` вместо «сырых» словарей.
- Генерация конфигов вынесена в отдельный класс `ConfigGenerator`.
- Утилитные функции `read_json_with_comments` / `write_json_with_header` устраняют дублирование кода чтения JSON с комментариями (было 4+ одинаковых блока).

#### Качество кода
- Добавлены type hints (Python 3.8+ с `from __future__ import annotations`).
- Константы вынесены в начало модуля — имена файлов, значения по умолчанию, допустимые сети.
- Убран `bare except` (observatory) — теперь все ошибки логируются.
- VLESS URL валидируется при добавлении, а не только при генерации — ошибка видна сразу.
- Условная генерация `realitySettings` — блок добавляется только если `security == "reality"`.
- Удалены «магические» дефолты для `pbk` и `flow` при пустых значениях — если параметр отсутствует в URL, он остаётся пустым (кроме `flow`, для которого по умолчанию `xtls-rprx-vision`).

#### UX
- Версия приложения отображается в заголовке окна.
- Удалён устаревший обработчик `on_proxy_selected` (был пустым `pass`).
- Информационная панель использует имена файлов из констант — при переименовании файлов меняется в одном месте.

## [3.0.0]

- Начальная публичная версия с загрузкой существующих конфигов.
